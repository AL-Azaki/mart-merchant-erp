# المواصفات الوظيفية المؤسسية لوحدة كتالوج المنتجات
**المشروع:** Smart Merchant ERP  
**النطاق:** Product Catalog Module  
**الإصدار:** 2.0 (النسخة المحسّنة والموحّدة)  
**الهدف:** توثيق شامل ومتكامل لكافة متطلبات، عمليات، سياسات، وقيود وحدة كتالوج المنتجات، لتكون مرجعاً هندسياً وتشغيلياً مغلقاً للمطورين ومحللي الأعمال.

---

## 1. الأهداف الإستراتيجية والنطاق

### 1.1 الأهداف
- توفير هيكل بيانات مرن وشامل لإدارة جميع أنواع المنتجات والخدمات التي تقدمها الشركة.
- تمكين التصنيف المتقدم والتجميع الذكي للمنتجات لدعم عمليات التسعير، الترويج، والتقارير.
- دعم تعدد السمات والخصائص للمنتجات (الألوان، المقاسات، المواد، وغيرها).
- ضمان تكامل سلس مع وحدات المخزون، المبيعات، المشتريات، والمحاسبة.
- توفير محرك بحث قوي وسريع لدعم عمليات البيع في نقاط البيع والتجارة الإلكترونية.

### 1.2 النطاق
يشمل النطاق إدارة هيكل المنتجات (الأصناف، التصنيفات، السمات، الوحدات)، إدارة الصور والملفات المرتبطة، إدارة الأسعار والتكاليف، إدارة حالة المنتج، وإدارة العلاقات بين المنتجات (المنتجات المرتبطة، المكملة، البديلة).

### 1.3 المسؤوليات الأساسية
- **إدارة البيانات:** الحفاظ على دقة واكتمال بيانات المنتجات.
- **التوحيد:** ضمان توحيد بيانات المنتجات عبر جميع الفروع والقنوات.
- **التكامل:** توفير بيانات المنتجات لجميع الوحدات الأخرى (المبيعات، المخزون، المشتريات، المحاسبة).
- **المرونة:** السماح بإضافة سمات وخصائص جديدة دون الحاجة إلى تغيير هيكل النظام الأساسي.

## 2. الأمان والصلاحيات (الحوكمة)

### 2.1 أدوار المستخدمين (User Roles)
| الدور | المسؤولية الأساسية |
| :--- | :--- |
| **محرر المنتجات (Product Editor)** | إنشاء وتعديل المنتجات، إدارة التصنيفات والسمات. |
| **مدير المنتجات (Product Manager)** | اعتماد المنتجات الجديدة، إدارة استراتيجيات التسعير، الموافقة على التغييرات الرئيسية. |
| **مشاهد المنتجات (Product Viewer)** | عرض بيانات المنتجات فقط (للبيع والمخزون والمشتريات). |

### 2.2 الصلاحيات الأساسية (Permissions)
- `Create_Product` (إنشاء منتج جديد)
- `Edit_Product` (تعديل منتج موجود)
- `Delete_Product` (حذف منتج - مقيد جداً)
- `Approve_Product` (اعتماد منتج للنشر)
- `Manage_Categories` (إدارة التصنيفات)
- `Manage_Attributes` (إدارة السمات)
- `Manage_Pricing` (إدارة الأسعار والتكاليف)
- `View_Cost_Price` (عرض سعر التكلفة - صلاحية مقيدة)

### 2.3 القواعد الأمنية (Security Rules)
- **منع حذف المنتجات:** يمنع حذف أي منتج تم استخدامه في أي حركة مخزنية أو مبيعات. بدلاً من ذلك، يتم تغيير حالة المنتج إلى "غير نشط" (Inactive).
- **سجل التدقيق:** تسجيل جميع التغييرات التي تطرأ على بيانات المنتج (من أنشأ، متى، ماذا غيّر).
- **إخفاء التكلفة:** إخفاء سعر التكلفة عن المستخدمين غير المصرح لهم (مثل مندوبي المبيعات والكاشير).
- **موافقة مزدوجة:** يتطلب تغيير السعر الأساسي أو التكلفة موافقة مدير المنتجات والمدير المالي.

## 3. هيكل المنتج الأساسي

### 3.1 التسلسل الهرمي للتصنيف (Category Hierarchy)
يُبنى هيكل التصنيف وفق التسلسل التالي:  
**القسم (Division)** ➔ **الفئة (Category)** ➔ **الفئة الفرعية (Sub-Category)** ➔ **المجموعة (Group)** ➔ **المنتج (Product)**

#### مستويات التصنيف:
| المستوى | الوصف | مثال |
| :--- | :--- | :--- |
| **القسم (Division)** | أعلى مستوى تصنيف. | إلكترونيات، ملابس، مواد غذائية. |
| **الفئة (Category)** | تصنيف رئيسي داخل القسم. | هواتف، أجهزة كمبيوتر، تلفزيونات. |
| **الفئة الفرعية (Sub-Category)** | تصنيف فرعي. | هواتف ذكية، هواتف تقليدية. |
| **المجموعة (Group)** | تجميع منتجات متشابهة. | هواتف سامسونج، هواتف آيفون. |
| **المنتج (Product)** | المنتج الفردي. | Samsung Galaxy S24 Ultra. |

### 3.2 البيانات الأساسية للمنتج (Core Product Data)
| الحقل | الوصف | إلزامي |
| :--- | :--- | :--- |
| **الرقم التعريفي (SKU)** | رقم فريد للمنتج (يتكون آلياً أو يدوياً). | نعم |
| **الباركود (Barcode)** | رمز شريطي للمنتج (UPC، EAN، أو أي نظام آخر). | نعم |
| **الاسم (Name)** | اسم المنتج (بالعربية والإنجليزية). | نعم |
| **الاسم المختصر (Short Name)** | للاستخدام في الإيصالات والفواتير المختصرة. | لا |
| **الوصف (Description)** | وصف تفصيلي للمنتج. | لا |
| **الوصف المختصر (Short Description)** | للاستخدام في المتجر الإلكتروني وبطاقات المنتج. | لا |
| **العلامة التجارية (Brand)** | ماركة المنتج. | لا |
| **المورد (Supplier)** | المورد الرئيسي للمنتج. | لا |
| **الوزن (Weight)** | وزن المنتج (للشحن). | لا |
| **الأبعاد (Dimensions)** | الطول، العرض، الارتفاع (للشحن). | لا |
| **حالة المنتج (Status)** | نشط، غير نشط، قيد المراجعة، موقف. | نعم |
| **نوع المنتج (Type)** | مادي، خدمي، رقمي، هدية. | نعم |

### 3.3 أنواع المنتجات (Product Types)
| النوع | الوصف | إدارة المخزون | مثال |
| :--- | :--- | :--- | :--- |
| **مادي (Physical)** | منتج ملموس له مخزون فعلي. | نعم | هاتف، كتاب، طعام. |
| **خدمي (Service)** | خدمة غير ملموسة (بدون مخزون). | لا | استشارات، اشتراكات، خدمات صيانة. |
| **رقمي (Digital)** | منتج رقمي قابل للتحميل. | لا | كتب إلكترونية، برامج، موسيقى. |
| **هدية (Gift)** | منتج يُباع كهدية (بطاقة هدايا). | نعم (رصيد إلكتروني) | بطاقة هدايا بقيمة 100 ريال. |
| **مركب (Composite/Bundle)** | مجموعة منتجات تباع كحزمة واحدة. | نعم (إدارة مكوناته) | حزمة لابتوب + حقيبة + ماوس. |

## 4. السمات والخصائص (Attributes & Variants)

### 4.1 السمات العامة (Global Attributes)
- **الهدف:** تحديد سمات مشتركة يمكن استخدامها عبر منتجات متعددة.
- **الأمثلة:** اللون، المقاس، المادة، الجنس، الموسم.
- **القيم:** يمكن أن تكون قائمة (مختارة من قيم محددة) أو نص حر (مثل الأبعاد، الوزن).

#### هيكل السمات:
| الحقل | الوصف | مثال |
| :--- | :--- | :--- |
| **اسم السمة (Attribute Name)** | اسم السمة. | اللون، المقاس، السعة. |
| **نوع السمة (Type)** | قائمة، رقم، نص، تاريخ، منطقي (نعم/لا). | قائمة، رقم. |
| **قيم السمة (Values)** | القيم المسموحة. | أحمر، أزرق، أخضر / S، M، L، XL. |
| **إلزامية (Required)** | هل السمة إلزامية للمنتج؟ | نعم/لا. |
| **قابلة للبحث (Searchable)** | هل يمكن البحث بهذه السمة؟ | نعم/لا. |
| **قابلة للتصفية (Filterable)** | هل يمكن التصفية بهذه السمة؟ | نعم/لا. |

### 4.2 المتغيرات (Variants)
- **الهدف:** إدارة المنتجات التي تأتي بعدة خيارات (مثل نفس المنتج بألوان أو مقاسات مختلفة).
- **الآلية:** المنتج الرئيسي (Parent) يحتوي على سمات مشتركة، ولكل متغير (Variant) سمات خاصة به.
- **مثال:** تي شيرت (المنتج الرئيسي) ➔ متغيرات (أحمر - M، أحمر - L، أزرق - M، أزرق - L).

#### هيكل المتغيرات:
| الحقل | الوصف |
| :--- | :--- |
| **المنتج الرئيسي (Parent SKU)** | رابط للمنتج الرئيسي. |
| **المتغيرات (Variants)** | قائمة من المتغيرات، كل متغير له سماته الخاصة. |
| **SKU فريد** | لكل متغير SKU خاص به (قد يكون مشتقاً من الرئيسي). |
| **باركود فريد** | لكل متغير باركود خاص به. |
| **سعر خاص** | يمكن أن يكون لكل متغير سعر مختلف. |
| **تكلفة خاصة** | يمكن أن يكون لكل متغير تكلفة مختلفة. |
| **مخزون خاص** | يمكن تتبع المخزون لكل متغير على حدة. |
| **صورة خاصة** | يمكن إرفاق صورة خاصة بكل متغير. |

### 4.3 مجموعات السمات (Attribute Groups)
- **الهدف:** تجميع السمات المتشابهة لتسهيل الإدارة.
- **مثال:** مجموعة "المواصفات التقنية" (المعالج، الذاكرة، الشاشة)، مجموعة "المواصفات الفيزيائية" (الوزن، الأبعاد، المادة).

## 5. التسعير والتكاليف

### 5.1 هيكل التسعير
| نوع السعر | الوصف | مثال |
| :--- | :--- | :--- |
| **سعر التكلفة (Cost Price)** | سعر شراء المنتج من المورد. | 500 ريال. |
| **سعر البيع الأساسي (Base Price)** | السعر الافتراضي للبيع. | 750 ريال. |
| **سعر البيع للجملة (Wholesale Price)** | سعر البيع لكميات الجملة. | 650 ريال. |
| **سعر البيع للتجزئة (Retail Price)** | سعر البيع في نقاط البيع. | 799 ريال. |
| **سعر خاص (Special Price)** | سعر لفترة محدودة أو لعملاء محددين. | 699 ريال (لمدة أسبوع). |
| **سعر العميل (Customer Price)** | سعر محدد لعميل معين (عبر قوائم الأسعار). | يحدد لكل عميل/مجموعة. |

### 5.2 استراتيجيات التسعير (Pricing Strategies)
| الاستراتيجية | الوصف |
| :--- | :--- |
| **ثابت (Fixed)** | سعر ثابت لا يتغير. |
| **ديناميكي (Dynamic)** | يتغير السعر بناءً على قواعد (مثل العرض والطلب، الوقت، الكمية). |
| **على أساس التكلفة (Cost-Plus)** | سعر البيع = التكلفة + هامش ربح محدد (نسبة أو مبلغ). |
| **على أساس المنافس (Competition-Based)** | تحديد السعر بناءً على أسعار المنافسين. |
| **تدرج الكمية (Tiered)** | سعر يختلف حسب الكمية المشتراة. |

### 5.3 قواعد التسعير (Pricing Rules)
- **الحد الأدنى للربح:** منع البيع بأقل من هامش ربح محدد (إلا بصلاحية خاصة).
- **الحد الأقصى للخصم:** تحديد الحد الأقصى للخصم المسموح به على المنتج.
- **تحديث الأسعار التلقائي:** تحديث أسعار البيع تلقائياً عند تغير سعر التكلفة.
- **الموافقة المزدوجة:** تغيير السعر الأساسي يتطلب موافقة مدير المنتجات والمدير المالي.

## 6. إدارة الصور والملفات

### 6.1 أنواع الملفات المدعومة
| النوع | الوصف |
| :--- | :--- |
| **الصور الرئيسية (Main Images)** | صورة المنتج الرئيسية (تظهر في البحث والكتالوج). |
| **صور إضافية (Additional Images)** | صور متعددة للمنتج من زوايا مختلفة. |
| **صور المتغيرات (Variant Images)** | صور خاصة بكل متغير (مثل تي شيرت أحمر). |
| **ملفات PDF (PDF Files)** | كتيبات، أدلة استخدام، أوراق مواصفات. |
| **فيديوهات (Videos)** | فيديوهات توضيحية أو ترويجية للمنتج. |

### 6.2 مواصفات الصور
| الخاصية | المواصفة |
| :--- | :--- |
| **الدقة (Resolution)** | 1500 × 1500 بكسل (أو أعلى). |
| **نوع الملف (Format)** | JPG، PNG، WEBP. |
| **الحجم الأقصى (Max Size)** | 10 ميجابايت لكل صورة. |
| **نسبة العرض (Aspect Ratio)** | 1:1 (مربع) للصورة الرئيسية. |
| **الخلفية (Background)** | خلفية بيضاء أو شفافة (يفضل). |
| **العلامة المائية (Watermark)** | اختياري (حسب سياسة الشركة). |

### 6.3 معالجة الصور (Image Processing)
- **رفع الصور (Upload):** دعم الرفع الفردي والجماعي.
- **اقتصاص (Crop):** أداة اقتصاص لضبط الصورة.
- **تحجيم تلقائي (Auto-Resize):** إنشاء نسخ متعددة بأحجام مختلفة (للإيصالات، المتجر الإلكتروني، التطبيق).
- **ضغط (Compression):** ضغط الصور لتحسين سرعة التحميل دون فقدان كبير في الجودة.

## 7. العلاقات بين المنتجات

### 7.1 أنواع العلاقات
| النوع | الوصف | مثال |
| :--- | :--- | :--- |
| **مرتبط (Related)** | منتجات ذات صلة (قد تهم العميل). | هاتف ➔ غطاء، شاحن، سماعة. |
| **مكمل (Upsell)** | منتج أعلى سعراً أو أفضل جودة. | هاتف أساسي ➔ هاتف متطور. |
| **بديل (Cross-sell)** | منتج بديل مشابه. | هاتف سامسونج ➔ هاتف آيفون. |
| **حزمة (Bundle)** | مجموعة منتجات تباع بسعر مخفض. | لابتوب + حقيبة + ماوس. |
| **ملحق (Accessory)** | منتج يُباع مع منتج رئيسي. | طابعة ➔ أحبار. |

### 7.2 إدارة الحزم (Bundle Management)
- **الهدف:** بيع مجموعة منتجات كمنتج واحد بسعر مخفض.
- **المكونات:** يمكن أن تتكون الحزمة من منتجات فردية أو حزم فرعية.
- **التسعير:** يمكن أن يكون سعر الحزمة أقل من مجموع أسعار المكونات.
- **المخزون:** يتم خصم مخزون كل مكون عند بيع الحزمة.
- **المرتجعات:** يمكن إرجاع الحزمة كاملة أو جزء منها (حسب السياسة).

## 8. إدارة حالة المنتج (Product Lifecycle)

### 8.1 حالات المنتج (Product Status)
| الحالة | الوصف | إمكانية البيع |
| :--- | :--- | :--- |
| **مسودة (Draft)** | منتج قيد الإنشاء والمراجعة. | لا |
| **قيد المراجعة (Under Review)** | في انتظار الموافقة. | لا |
| **نشط (Active)** | منتج متاح للبيع. | نعم |
| **غير نشط (Inactive)** | منتج غير متاح للبيع مؤقتاً. | لا |
| **موقف (Suspended)** | تم إيقاف المنتج لأسباب تجارية أو قانونية. | لا |
| **متوقف (Discontinued)** | تم إيقاف المنتج نهائياً. | لا |
| **مؤرشف (Archived)** | تم أرشفة المنتج بعد إيقافه. | لا |

### 8.2 دورة حياة المنتج (Product Lifecycle)
1.  **إنشاء (Create):** إدخال بيانات المنتج الأساسية.
2.  **مراجعة (Review):** مراجعة البيانات من قبل مدير المنتجات.
3.  **اعتماد (Approve):** الموافقة على المنتج ونشره.
4.  **نشط (Active):** المنتج متاح للبيع.
5.  **تعديل (Modify):** تحديث بيانات المنتج (السعر، الوصف، الصور).
6.  **إيقاف مؤقت (Suspend):** إيقاف البيع مؤقتاً.
7.  **إيقاف دائم (Discontinue):** إيقاف البيع نهائياً.
8.  **أرشفة (Archive):** نقل المنتج إلى الأرشيف.

## 9. البحث والتصفية (Search & Filtering)

### 9.1 محرك البحث (Search Engine)
- **البحث النصي الكامل (Full-Text Search):** البحث في الاسم، الوصف، الكلمات المفتاحية.
- **البحث بالباركود:** البحث المباشر بالباركود أو SKU.
- **البحث الصوتي:** دعم البحث الصوتي (للتطبيقات).
- **البحث المتقدم:** البحث باستخدام سمات متعددة (اللون، المقاس، السعر، العلامة التجارية).
- **اقتراحات البحث (Search Suggestions):** اقتراح منتجات أثناء الكتابة.

### 9.2 التصفية (Filtering)
- **التصفية حسب السعر:** نطاق سعري (من - إلى).
- **التصفية حسب السمة:** اللون، المقاس، المادة، العلامة التجارية.
- **التصفية حسب التصنيف:** القسم، الفئة، الفئة الفرعية.
- **التصفية حسب الحالة:** المتاح، غير المتاح، الأكثر مبيعاً.
- **التصفية حسب التقييم:** تقييم العملاء.

### 9.3 الترتيب (Sorting)
- **الأكثر صلة:** حسب درجة المطابقة للبحث.
- **الأحدث:** حسب تاريخ الإضافة.
- **الأعلى سعراً:** تنازلي.
- **الأقل سعراً:** تصاعدي.
- **الأكثر مبيعاً:** حسب كمية المبيعات.
- **الأعلى تقييماً:** حسب تقييم العملاء.

## 10. استيراد وتصدير البيانات (Import/Export)

### 10.1 الاستيراد (Import)
- **مصادر الاستيراد:** ملفات Excel (XLSX، CSV)، ملفات XML، JSON.
- **التحقق من البيانات:** التحقق من صحة البيانات قبل الاستيراد (SKU فريد، تصنيفات صحيحة، وحدات قياس صحيحة).
- **خيارات الاستيراد:**
    - **إنشاء جديد:** إنشاء منتجات جديدة فقط.
    - **تحديث:** تحديث المنتجات الموجودة.
    - **إنشاء وتحديث:** إنشاء الجديد وتحديث الموجود.
- **معالجة الأخطاء:** عرض تقرير بالأخطاء (ما هي الصفوف التي فشلت ولماذا).
- **الاستيراد الجماعي:** دعم استيراد آلاف المنتجات دفعة واحدة.

### 10.2 التصدير (Export)
- **تنسيقات التصدير:** Excel (XLSX)، CSV، PDF، XML، JSON.
- **خيارات التصدير:**
    - **جميع الحقول:** تصدير كافة بيانات المنتج.
    - **حقول محددة:** اختيار الحقول المراد تصديرها.
    - **تصفية:** تصدير منتجات محددة (حسب التصنيف، الحالة، العلامة التجارية).
- **الجدولة:** إمكانية جدولة تصدير تلقائي (يومي، أسبوعي، شهري).

---

## 11. التكامل مع الوحدات الأخرى (Integration)

### 11.1 تكامل المخزون (Inventory Integration)
- توفير بيانات المنتج (SKU، باركود، وحدة القياس) لوحدة المخزون.
- مزامنة حالة المنتج مع وحدة المخزون (نشط ➔ يسمح بالحركة، غير نشط ➔ يمنع الحركة).
- تحديث التكلفة من وحدة المشتريات إلى كتالوج المنتجات.

### 11.2 تكامل المبيعات (Sales Integration)
- توفير بيانات المنتج (الاسم، السعر، الصورة) لوحدة المبيعات ونقاط البيع.
- دعم البحث عن المنتجات في واجهة البيع.
- تحديث الأسعار في نقاط البيع عند تغييرها في الكتالوج.

### 11.3 تكامل المشتريات (Procurement Integration)
- ربط المنتج بالموردين الرئيسيين والبدلاء.
- توفير بيانات المنتج لأوامر الشراء.
- تحديث سعر التكلفة عند استلام بضاعة جديدة.

### 11.4 تكامل التجارة الإلكترونية (E-Commerce Integration)
- تصدير بيانات المنتجات (الاسم، الوصف، الصور، السعر، السمات) إلى المتجر الإلكتروني.
- مزامنة المخزون والأسعار بين النظام والمتجر الإلكتروني.
- دعم تنسيقات البيانات الشائعة (مثل Google Shopping Feed).

## 12. التقارير ولوحات المعلومات (Analytics & Reporting)

### 12.1 التقارير الأساسية
| التقرير | الغرض |
| :--- | :--- |
| **تقرير المنتجات (Product Report)** | عرض كافة المنتجات مع بياناتها الأساسية. |
| **تقرير أسعار المنتجات (Pricing Report)** | عرض أسعار البيع والتكلفة لكل منتج. |
| **تقرير المنتجات غير النشطة (Inactive Products)** | عرض المنتجات غير النشطة أو الموقفة. |
| **تقرير المنتجات بدون حركة (No Movement)** | المنتجات التي لم تشهد أي حركة مبيعات أو مخزون. |

### 12.2 لوحات المعلومات (Dashboards)
- **لوحة المنتجات:** عرض إجمالي عدد المنتجات، عدد المنتجات النشطة، عدد التصنيفات، عدد العلامات التجارية.
- **لوحة الأسعار:** عرض توزيع الأسعار، أعلى وأقل 10 منتجات سعراً.
- **لوحة التصنيفات:** عرض توزيع المنتجات حسب التصنيفات.

### 12.3 مؤشرات الأداء (KPIs)
| المؤشر | الوصف |
| :--- | :--- |
| **عدد المنتجات النشطة** | إجمالي المنتجات المتاحة للبيع. |
| **نسبة المنتجات ذات الصور** | النسبة المئوية للمنتجات التي تحتوي على صور. |
| **متوسط سعر البيع** | متوسط سعر بيع المنتجات النشطة. |
| **تنوع المنتجات** | عدد التصنيفات والعلامات التجارية. |

## 13. العمل في وضع الأوفلاين (Offline Mode)

### 13.1 السلوك في وضع الأوفلاين
- **نقاط البيع (POS):** تعمل بالنسخة المخزنة محلياً من كتالوج المنتجات.
- **التحديثات:** يتم تحديث الكتالوج محلياً عند عودة الاتصال.
- **المنتجات الجديدة:** يمكن إضافة منتجات جديدة في وضع الأوفلاين، وتنتظر المزامنة.

### 13.2 المزامنة (Synchronization)
- **تنزيل الكتالوج:** تنزيل تحديثات الكتالوج (المنتجات الجديدة، التحديثات، الحذف) عند عودة الاتصال.
- **رفع التغييرات:** رفع المنتجات الجديدة أو التعديلات التي تمت في وضع الأوفلاين.
- **سياسة النزاعات:** Server-Wins للبيانات الأساسية (التصنيفات، السمات، العلاقات).

## 14. التوسعات المستقبلية (Future Extensions)

- **الواقع المعزز (AR):** دعم عرض المنتجات بتقنية الواقع المعزز في التطبيق.
- **الذكاء الاصطناعي (AI):** اقتراح منتجات تلقائياً، تحليل بيانات المنتجات، التنبؤ بالطلب.
- **إدارة المنتجات متعددة اللغات:** دعم بيانات المنتج بلغات متعددة (العربية، الإنجليزية، الفرنسية).
- **تخصيص المنتج (Customization):** السماح للعملاء بتخصيص المنتجات (النقش، الألوان).

## 15. متطلبات الأداء (Performance Requirements)

- **البحث عن منتج:** يجب أن يستجيب في أقل من 200 مللي ثانية لقاعدة بيانات تحتوي على مليون منتج.
- **تحميل صفحة المنتج:** أقل من 1 ثانية (مع الصور).
- **استيراد المنتجات:** معالجة 10,000 منتج في أقل من 5 دقائق.
- **تحديث المنتج:** يجب أن يتم في أقل من 500 مللي ثانية.
- **مزامنة الكتالوج:** يجب أن تتم في الخلفية دون تعطيل المستخدم.

## 16. سيناريوهات الإجهاد المؤسسية (Enterprise Stress Scenarios)

| السيناريو | آلية المواجهة |
| :--- | :--- |
| **مليون منتج في الكتالوج** | استخدام الفهارس المناسبة، التقسيم (Partitioning)، والتخزين المؤقت (Caching). |
| **آلاف الطلبات المتزامنة للبحث** | استخدام خوادم بحث مخصصة (Elasticsearch)، وتوزيع الأحمال. |
| **تحديثات متزامنة للكتالوج** | استخدام القفل المتفائل (Optimistic Locking) لمنع التصادمات. |
| **استيراد ضخم للمنتجات** | معالجة الدُفعات (Batch Processing)، واستخدام قوائم الانتظار (Queues). |
| **فشل الخادم المركزي** | استخدام النسخ الاحتياطي (Replicas) وخطط التعافي من الكوارث (Disaster Recovery). |
