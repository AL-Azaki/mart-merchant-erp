# المواصفات الوظيفية المؤسسية لوحدة التقارير والذكاء الأعمالي (BI)
**المشروع:** Smart Merchant ERP  
**النطاق:** Reporting & Business Intelligence Module  
**الإصدار:** 2.0 (النسخة المحسّنة والموحّدة)  
**الهدف:** توثيق شامل ومتكامل لكافة متطلبات، عمليات، سياسات، وقيود وحدة التقارير والذكاء الأعمالي، لتكون مرجعاً هندسياً وتشغيلياً مغلقاً للمطورين ومحللي الأعمال.

---

## 1. الأهداف الإستراتيجية والنطاق

### 1.1 الأهداف
- توفير نظام تقارير وتحليلات متكامل يدعم اتخاذ القرارات الاستراتيجية والتشغيلية.
- تمكين المستخدمين من الوصول إلى البيانات وتحليلها بسهولة وسرعة.
- توفير رؤية شاملة لأداء الشركة من خلال لوحات معلومات تفاعلية.
- دعم التحليل متعدد الأبعاد (OLAP) لتحديد الاتجاهات والأنماط.
- ضمان دقة وموثوقية البيانات المستخدمة في التقارير والتحليلات.

### 1.2 النطاق
يشمل النطاق إدارة التقارير (إنشاء، تخصيص، جدولة، تصدير)، لوحات المعلومات التفاعلية (Dashboards)، تحليل البيانات متعدد الأبعاد، مؤشرات الأداء الرئيسية (KPIs)، التنبؤ والتحليل التنبؤي، استخراج البيانات (Data Mining)، والتكامل مع جميع وحدات النظام للحصول على البيانات.

### 1.3 المسؤوليات الأساسية
- **دقة البيانات:** ضمان دقة وموثوقية البيانات المستخدمة في التقارير.
- **توقيت البيانات:** ضمان تحديث البيانات في الوقت المناسب.
- **سهولة الوصول:** توفير وصول سهل وبديهي للبيانات والتقارير.
- **الأمان:** ضمان أمان البيانات وسريتها من خلال التحكم في الوصول.
- **الدعم:** تقديم الدعم للمستخدمين في إنشاء التقارير وتحليل البيانات.

## 2. الأمان والصلاحيات (الحوكمة)

### 2.1 أدوار المستخدمين (User Roles)
| الدور | المسؤولية الأساسية |
| :--- | :--- |
| **مستخدم تقارير (Report User)** | عرض التقارير ولوحات المعلومات المصرح له بها. |
| **محلل أعمال (Business Analyst)** | إنشاء وتعديل التقارير ولوحات المعلومات. |
| **مدير التقارير (Report Manager)** | إدارة التقارير، التحكم في الوصول، جدولة التقارير. |
| **مسؤول النظام (System Admin)** | إدارة إعدادات النظام، مستودع البيانات، تكامل البيانات. |
| **مدير تنفيذي (Executive)** | عرض لوحات المعلومات التنفيذية، التقارير الاستراتيجية. |

### 2.2 الصلاحيات الأساسية (Permissions)
- `View_Report` (عرض تقرير)
- `Create_Report` (إنشاء تقرير جديد)
- `Edit_Report` (تعديل تقرير)
- `Delete_Report` (حذف تقرير - مقيد جداً)
- `Schedule_Report` (جدولة تقرير)
- `Export_Report` (تصدير تقرير)
- `Create_Dashboard` (إنشاء لوحة معلومات جديدة)
- `Edit_Dashboard` (تعديل لوحة معلومات)
- `Delete_Dashboard` (حذف لوحة معلومات)
- `Manage_Data_Sources` (إدارة مصادر البيانات)
- `View_Sensitive_Data` (عرض بيانات حساسة - صلاحية مقيدة)
- `Manage_Users_Permissions` (إدارة صلاحيات المستخدمين على التقارير)

### 2.3 القواعد الأمنية (Security Rules)
- **الوصول على أساس الدور:** تحديد صلاحيات الوصول للتقارير ولوحات المعلومات بناءً على أدوار المستخدمين.
- **الوصول على أساس البيانات:** تحديد الوصول إلى البيانات بناءً على مستوى المستخدم (مثل: مدير فرع يرى بيانات فرعه فقط).
- **تشفير البيانات:** تشفير البيانات الحساسة في التقارير.
- **سجل التدقيق:** تسجيل كل عملية وصول، إنشاء، تعديل، أو تصدير للتقارير.
- **إخفاء البيانات الحساسة:** إخفاء البيانات الحساسة (مثل الرواتب، المعلومات الشخصية) في التقارير للمستخدمين غير المصرح لهم.

## 3. مستودع البيانات (Data Warehouse)

### 3.1 هيكل مستودع البيانات
| المكون | الوصف |
| :--- | :--- |
| **طبقة التجميع (Staging Layer)** | استخراج البيانات من المصادر المختلفة (OLTP) وتنظيفها وتحويلها. |
| **طبقة البيانات الأساسية (Core Layer)** | تخزين البيانات المحولة والمجمعة في نموذج أبعاد (Star Schema / Snowflake Schema). |
| **طبقة التجميع (Aggregation Layer)** | تجميع البيانات لمستويات مختلفة (يومي، أسبوعي، شهري، سنوي). |
| **طبقة العرض (Presentation Layer)** | توفير البيانات للمستخدمين النهائيين عبر التقارير ولوحات المعلومات. |

### 3.2 نماذج البيانات (Data Models)
| النموذج | الوصف |
| :--- | :--- |
| **نجمي (Star Schema)** | جدول حقائق (Fact Table) مركزي محاط بجداول أبعاد (Dimension Tables). |
| **ثلجي (Snowflake Schema)** | تطوير للنموذج النجمي مع تطبيع إضافي لجداول الأبعاد. |
| **مكعب (Cube)** | نموذج متعدد الأبعاد لتحليل OLAP (Online Analytical Processing). |

### 3.3 جداول الحقائق الأساسية (Fact Tables)
| الجدول | الوصف | الأبعاد المرتبطة |
| :--- | :--- | :--- |
| **مبيعات (Sales Fact)** | حقائق المبيعات (الكمية، السعر، الخصم، الضريبة). | الوقت، المنتج، العميل، الفرع، الموظف. |
| **مشتريات (Purchases Fact)** | حقائق المشتريات (الكمية، السعر، الضريبة). | الوقت، المنتج، المورد، الفرع. |
| **مخزون (Inventory Fact)** | حقائق المخزون (الكمية، التكلفة). | الوقت، المنتج، المستودع. |
| **مالية (Finance Fact)** | حقائق مالية (الإيرادات، المصروفات، الأصول، الخصوم). | الوقت، الحساب، الفرع. |
| **موارد بشرية (HR Fact)** | حقائق الموارد البشرية (الرواتب، الحضور، الإجازات). | الوقت، الموظف، القسم. |

### 3.4 جداول الأبعاد الأساسية (Dimension Tables)
| البعد | الوصف | السمات |
| :--- | :--- | :--- |
| **الوقت (Time)** | أبعاد الوقت (سنة، شهر، أسبوع، يوم). | السنة، الربع، الشهر، الأسبوع، اليوم. |
| **المنتج (Product)** | أبعاد المنتج. | الفئة، الفئة الفرعية، العلامة التجارية، المورد. |
| **العميل (Customer)** | أبعاد العميل. | النوع، المنطقة، المجموعة، التصنيف. |
| **المورد (Supplier)** | أبعاد المورد. | التصنيف، المنطقة، التقييم. |
| **الفرع (Branch)** | أبعاد الفرع. | المنطقة، المدينة، النوع. |
| **الموظف (Employee)** | أبعاد الموظف. | القسم، الوظيفة، المشرف. |
| **الحساب (Account)** | أبعاد الحساب المحاسبي. | النوع، الفئة، المجموعة. |

| النوع | الوصف | التكرار |

| **تقرير أداء الفروع (Branch Performance)** | مقارنة أداء الفروع المختلفة. | شهري / ربع سنوي |

| النوع | الوصف | التكرار |
| :--- | :--- | :--- |

| **تحليل الربحية (Profitability Analysis)** | تحليل الربحية حسب المنتج، العميل، المنطقة. | ربع سنوي / سنوي |
| **تحليل حصة السوق (Market Share Analysis)** | تحليل حصة الشركة في السوق. | سنوي |
| **تحليل SWOT** | تحليل نقاط القوة والضعف والفرص والتهديدات. | سنوي |

## 5. لوحات المعلومات (Dashboards)

### 5.1 أنواع لوحات المعلومات
| النوع | الوصف | الجمهور |
| :--- | :--- | :--- |
| **لوحة تنفيذية (Executive Dashboard)** | عرض مؤشرات الأداء الرئيسية للشركة. | الإدارة التنفيذية |
| **لوحة إدارية (Managerial Dashboard)** | عرض مؤشرات الأداء للإدارة الوسطى. | المديرين |
| **لوحة تشغيلية (Operational Dashboard)** | عرض مؤشرات الأداء التشغيلية (يومية). | المشرفين والموظفين |
| **لوحة تحليلية (Analytical Dashboard)** | عرض تحليلات متعمقة للبيانات. | محللي الأعمال |

### 5.2 مكونات لوحة المعلومات
| المكون | الوصف |
| :--- | :--- |
| **بطاقة KPI (KPI Card)** | عرض قيمة مؤشر أداء رئيسي واحد. |
| **رسم بياني (Chart)** | عرض البيانات في شكل رسوم بيانية (شريطي، خطي، دائري، مساحي). |
| **جدول (Table)** | عرض البيانات في شكل جدول. |
| **خريطة (Map)** | عرض البيانات على خريطة جغرافية. |
| **مؤشر (Gauge)** | عرض قيمة مؤشر في شكل عداد. |
| **جدول زمني (Timeline)** | عرض الأحداث على جدول زمني. |
| **مرشحات (Filters)** | تصفية البيانات المعروضة في لوحة المعلومات. |
| **عناصر تحكم (Controls)** | عناصر تحكم للتفاعل مع البيانات (أزرار، قوائم منسدلة). |

### 5.3 أمثلة على لوحات المعلومات
- **لوحة المبيعات التنفيذية:**
    - إجمالي المبيعات (KPI).
    - المبيعات حسب الشهر (رسم بياني خطي).
    - أفضل المنتجات مبيعاً (رسم بياني شريطي).
    - المبيعات حسب المنطقة (خريطة).
    - نمو المبيعات (مؤشر Gauge).

- **لوحة المخزون التشغيلية:**
    - قيمة المخزون الإجمالية (KPI).
    - معدل دوران المخزون (KPI).
    - المنتجات الأقل في المخزون (جدول).
    - حركة المخزون (رسم بياني خطي).
    - المنتجات منتهية الصلاحية (تنبيه).

## 6. تحليل البيانات (Data Analysis)

### 6.1 تقنيات التحليل
| التقنية | الوصف |
| :--- | :--- |
| **تحليل الاتجاهات (Trend Analysis)** | تحليل البيانات على مدى فترة زمنية لتحديد الاتجاهات. |
| **التحليل المقارن (Comparative Analysis)** | مقارنة البيانات بين فترات أو مجموعات مختلفة. |
| **التحليل النسبي (Ratio Analysis)** | حساب النسب المالية والتشغيلية. |
| **تحليل التباين (Variance Analysis)** | مقارنة الأداء الفعلي بالمخطط (ميزانية). |
| **تحليل التوزيع (Distribution Analysis)** | تحليل توزيع البيانات (مثل توزيع المبيعات حسب الفئة). |
| **تحليل الارتباط (Correlation Analysis)** | تحديد العلاقة بين متغيرين أو أكثر. |
| **تحليل الانحدار (Regression Analysis)** | تحديد العلاقة بين متغير تابع ومتغيرات مستقلة. |
| **التحليل العنقودي (Cluster Analysis)** | تجميع البيانات في مجموعات متشابهة (تجزئة العملاء). |

### 6.2 التحليل متعدد الأبعاد (OLAP)
| العملية | الوصف |
| :--- | :--- |
| **تقطيع (Slicing)** | اختيار بعد واحد من المكعب. |
| **تقطيع شرائح (Dicing)** | اختيار مجموعة فرعية من المكعب (بعدين أو أكثر). |
| **تنقيب (Drill-Down)** | الانتقال من مستوى عالٍ من التجميع إلى مستوى أدنى. |
| **رفع (Roll-Up)** | الانتقال من مستوى منخفض إلى مستوى أعلى من التجميع. |
| **تدوير (Pivot)** | إعادة ترتيب الأبعاد في المكعب. |

## 7. التنبؤ والتحليل التنبؤي (Forecasting & Predictive Analytics)

### 7.1 تقنيات التنبؤ
| التقنية | الوصف |
| :--- | :--- |
| **المتوسط المتحرك (Moving Average)** | التنبؤ بناءً على متوسط البيانات السابقة. |
| **التجانس الأسي (Exponential Smoothing)** | التنبؤ بإعطاء وزن أكبر للبيانات الحديثة. |
| **تحليل الاتجاه (Trend Analysis)** | التنبؤ بناءً على الاتجاه التاريخي. |
| **الانحدار الخطي (Linear Regression)** | التنبؤ باستخدام علاقة خطية بين المتغيرات. |
| **الانحدار المتعدد (Multiple Regression)** | التنبؤ باستخدام علاقة بين متغيرات متعددة. |
| **نماذج ARIMA** | نماذج متقدمة للتنبؤ بالسلاسل الزمنية. |
| **الشبكات العصبية (Neural Networks)** | التنبؤ باستخدام الذكاء الاصطناعي. |

### 7.2 مجالات التنبؤ
| المجال | الوصف |
| :--- | :--- |
| **تنبؤ المبيعات (Sales Forecasting)** | التنبؤ بالمبيعات المستقبلية. |
| **تنبؤ الطلب (Demand Forecasting)** | التنبؤ بالطلب على المنتجات. |
| **تنبؤ التدفق النقدي (Cash Flow Forecasting)** | التنبؤ بالتدفقات النقدية المستقبلية. |
| **تنبؤ المخزون (Inventory Forecasting)** | التنبؤ باحتياجات المخزون. |
| **تنبؤ الموارد البشرية (HR Forecasting)** | التنبؤ باحتياجات الموظفين ومعدل الدوران. |

## 8. استخراج البيانات (Data Mining)

### 8.1 تقنيات استخراج البيانات
| التقنية | الوصف |
| :--- | :--- |
| **التصنيف (Classification)** | تصنيف البيانات إلى فئات محددة مسبقاً. |
| **التجميع (Clustering)** | تجميع البيانات في مجموعات متشابهة. |
| **قواعد الارتباط (Association Rules)** | اكتشاف العلاقات بين العناصر (مثال: العملاء الذين يشترون X يشترون Y). |
| **الكشف عن الشذوذ (Anomaly Detection)** | اكتشاف البيانات غير الطبيعية أو الاستثنائية. |
| **الانحدار (Regression)** | التنبؤ بالقيم الرقمية. |
| **التجميع الهرمي (Hierarchical Clustering)** | تجميع البيانات في هيكل هرمي. |

### 8.2 تطبيقات استخراج البيانات
| التطبيق | الوصف |
| :--- | :--- |
| **تجزئة العملاء (Customer Segmentation)** | تقسيم العملاء إلى مجموعات متشابهة. |
| **تحليل سلة السوق (Market Basket Analysis)** | تحليل المنتجات التي يتم شراؤها معاً. |
| **كشف الاحتيال (Fraud Detection)** | اكتشاف المعاملات المشبوهة. |
| **توصية المنتجات (Product Recommendation)** | اقتراح منتجات للعملاء بناءً على سلوكهم. |
| **تحليل الأسباب الجذرية (Root Cause Analysis)** | تحديد الأسباب الجذرية للمشكلات. |

## 10. تصدير التقارير (Report Export)

### 10.1 تنسيقات التصدير المدعومة
| التنسيق | الوصف |
| :--- | :--- |
| **PDF** | تنسيق المستندات المحمول (للطباعة والمشاركة). |

| **Word (DOCX)** | تنسيق مستندات Word. |
| **HTML** | تنسيق صفحات الويب. |
| **JSON** | تنسيق تبادل البيانات. |
| **XML** | تنسيق الترميز الموسع. |
| **صورة (Image)** | تنسيق الصور (PNG، JPG). |

## 11. التكامل مع الوحدات الأخرى (Integration)

### 11.1 مصادر البيانات
| المصدر | الوصف |
| :--- | :--- |
| **المبيعات (Sales)** | بيانات المبيعات والفواتير والعملاء. |
| **المشتريات (Procurement)** | بيانات المشتريات والموردين وأوامر الشراء. |
| **المخزون (Inventory)** | بيانات المخزون والمنتجات والمستودعات. |
| **المالية (Finance)** | بيانات القيود المحاسبية والحسابات. |
| **الموارد البشرية (HR)** | بيانات الموظفين والحضور والرواتب. |
| **الأصول الثابتة (Fixed Assets)** | بيانات الأصول الثابتة والإهلاك. |
| **إدارة المستندات (Document Management)** | بيانات المستندات والملفات. |
| **إدارة المشاريع (Project Management)** | بيانات المشاريع والمهام والموارد. |

### 11.2 تكامل البيانات
- **ETL (Extract, Transform, Load):** استخراج البيانات من المصادر، تحويلها، وتحميلها في مستودع البيانات.
- **تحديث البيانات:** تحديث البيانات في مستودع البيانات بشكل دوري (يومي، أسبوعي، شهري).
- **تحديث فوري (Real-time):** تحديث البيانات في الوقت الفعلي (لللوحات التشغيلية).
- **مصادر بيانات خارجية:** استيراد البيانات من مصادر خارجية (مثل بيانات السوق، بيانات المنافسين).

## 12. العمل في وضع الأوفلاين (Offline Mode)

### 12.1 السلوك في وضع الأوفلاين
- **عرض التقارير:** عرض التقارير المخزنة محلياً (قد تكون غير محدثة).
- **لوحات المعلومات:** عرض لوحات المعلومات المخزنة محلياً.
- **تصدير التقارير:** يمكن تصدير التقارير المخزنة محلياً.
- **إنشاء التقارير:** لا يمكن إنشاء تقارير جديدة في وضع الأوفلاين (تتطلب اتصالاً بمستودع البيانات).

### 12.2 المزامنة (Synchronization)
- **تنزيل البيانات:** تنزيل التقارير ولوحات المعلومات المحدثة من الخادم المركزي.
- **سياسة النزاعات:** Server-Wins للتقارير ولوحات المعلومات (الخادم هو المصدر الأساسي).

## 15. متطلبات الأداء (Performance Requirements)

- **عرض تقرير:** أقل من 3 ثوانٍ (لبيانات حتى 100,000 سجل).
- **عرض لوحة معلومات:** أقل من 5 ثوانٍ.
- **تصدير تقرير:** أقل من 10 ثوانٍ (لـ 10,000 سجل).
- **تحديث البيانات:** تحديث مستودع البيانات في أقل من ساعة (لمليون سجل).
- **تنفيذ استعلام تحليلي:** أقل من 5 ثوانٍ.

## 16. سيناريوهات الإجهاد المؤسسية (Enterprise Stress Scenarios)

| السيناريو | آلية المواجهة |
| :--- | :--- |
| **ملايين السجلات في مستودع البيانات** | استخدام التقسيم (Partitioning)، الفهارس المحسّنة، التخزين المؤقت (Caching). |
| **آلاف المستخدمين يصلون للتقارير في نفس الوقت** | استخدام خوادم متعددة (Load Balancing)، التوزيع الجغرافي (CDN). |
| **تحديث البيانات أثناء ساعات الذروة** | جدولة تحديث البيانات في أوقات منخفضة الضغط (خارج ساعات العمل). |
| **فشل الخادم المركزي** | استخدام النسخ الاحتياطي (Replicas) وخطط التعافي من الكوارث (Disaster Recovery). |
| **استعلامات تحليلية معقدة** | استخدام نماذج بيانات محسّنة، التجميع المسبق (Pre-aggregation). |
