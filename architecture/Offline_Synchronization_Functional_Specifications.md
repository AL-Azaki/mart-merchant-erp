# المواصفات الوظيفية المؤسسية لوحدة العمل دون اتصال والمزامنة
**المشروع:** Smart Merchant ERP  
**النطاق:** Offline & Synchronization Module  
**الإصدار:** 2.0 (النسخة المحسّنة والموحّدة)  
**الهدف:** توثيق شامل ومتكامل لكافة متطلبات، عمليات، سياسات، وقيود وحدة العمل دون اتصال والمزامنة، لتكون مرجعاً هندسياً وتشغيلياً مغلقاً للمطورين ومحللي الأعمال.

---

## 1. الأهداف الإستراتيجية والنطاق

### 1.1 الأهداف
- ضمان استمرارية الأعمال في حالة انقطاع الاتصال بالإنترنت أو الخادم المركزي.
- تمكين المستخدمين من العمل بكامل طاقتهم في وضع الأوفلاين دون فقدان الإنتاجية.
- ضمان مزامنة آمنة وموثوقة للبيانات بين الأجهزة المحلية والخادم المركزي.
- منع فقدان البيانات أو تضاربها أثناء عملية المزامنة.
- توفير تجربة مستخدم سلسة وشفافة (يعمل النظام بشكل طبيعي دون أن يشعر المستخدم بفقدان الاتصال).

### 1.2 النطاق
يشمل النطاق إدارة البيانات المحلية (Local Database)، إدارة طابور المزامنة (Sync Queue)، معالجة التعارضات (Conflict Resolution)، إدارة سياسات المزامنة، إدارة الأوفلاين لكل وحدة من وحدات النظام، وإدارة استرداد البيانات بعد انقطاع الاتصال.

### 1.3 المسؤوليات الأساسية
- **استمرارية الأعمال:** ضمان استمرار العمل في جميع الظروف.
- **سلامة البيانات:** ضمان عدم فقدان أو تلف البيانات أثناء الأوفلاين والمزامنة.
- **الاتساق:** ضمان اتساق البيانات بين الأجهزة المحلية والخادم المركزي.
- **الشفافية:** توفير تجربة مستخدم سلسة دون تعقيدات تقنية.
- **التعافي:** ضمان استرداد البيانات بشكل كامل بعد عودة الاتصال.

## 2. الأمان والصلاحيات (الحوكمة)

### 2.1 أدوار المستخدمين (User Roles)
| الدور | المسؤولية الأساسية |
| :--- | :--- |
| **مستخدم (User)** | استخدام النظام في وضع الأوفلاين دون القلق بشأن التفاصيل التقنية. |
| **مشرف (Supervisor)** | مراقبة حالة المزامنة لفريقه، التدخل في حالة وجود تعارضات. |
| **مدير النظام (System Admin)** | إدارة إعدادات الأوفلاين والمزامنة، مراقبة حالة الخوادم، حل مشكلات المزامنة. |
| **مدير تكنولوجيا المعلومات (IT Manager)** | إدارة البنية التحتية، التخزين المحلي، سياسات الأمان. |

### 2.2 الصلاحيات الأساسية (Permissions)
- `View_Sync_Status` (عرض حالة المزامنة)
- `View_Offline_Data` (عرض البيانات المحلية)
- `Force_Sync` (فرض المزامنة - صلاحية مقيدة)
- `Clear_Sync_Queue` (مسح طابور المزامنة - صلاحية مقيدة جداً)
- `Manage_Sync_Policies` (إدارة سياسات المزامنة)
- `View_Conflict_Log` (عرض سجل التعارضات)
- `Resolve_Conflict` (حل التعارضات - صلاحية مقيدة)

### 2.3 القواعد الأمنية (Security Rules)
- **تشفير البيانات المحلية:** تشفير جميع البيانات المخزنة محلياً على الأجهزة (لمنع الوصول غير المصرح به في حالة فقدان الجهاز).
- **مصادقة الجهاز:** التأكد من هوية الجهاز قبل السماح بالمزامنة.
- **سجل التدقيق:** تسجيل جميع عمليات المزامنة والبيانات المنقولة.
- **الحد من حجم البيانات:** تحديد الحد الأقصى لحجم البيانات التي يمكن تخزينها محلياً.
- **التحقق من سلامة البيانات:** التحقق من سلامة البيانات قبل وبعد المزامنة (باستخدام Checksums أو Hashing).

## 3. البنية التقنية للأوفلاين (Offline Architecture)

### 3.1 مكونات البنية
| المكون | الوصف |
| :--- | :--- |
| **قاعدة البيانات المحلية (Local DB)** | قاعدة بيانات مخزنة على الجهاز المحلي (SQLite، IndexedDB، أو ما شابه). |
| **طابور المزامنة (Sync Queue)** | طابور يخزن العمليات التي تحتاج إلى مزامنة (إدراج، تحديث، حذف). |
| **مدير الأوفلاين (Offline Manager)** | يدير عمليات الأوفلاين (التخزين، الاسترجاع، المزامنة). |
| **مدير التعارضات (Conflict Manager)** | يدير تعارضات البيانات بين المحلي والمركزي. |
| **محرك المزامنة (Sync Engine)** | ينفذ عمليات المزامنة (رفع وتنزيل البيانات). |
| **ذاكرة التخزين المؤقت (Cache)** | تخزين البيانات المستخدمة بشكل متكرر لتحسين الأداء. |

### 3.2 استراتيجيات المزامنة
| الاستراتيجية | الوصف |
| :--- | :--- |
| **Client-Wins** | بيانات العميل (المحلية) هي الأصح في حالة التعارض. |
| **Server-Wins** | بيانات الخادم (المركزية) هي الأصح في حالة التعارض. |
| **Last-Write-Wins** | التعديل الأحدث هو الأصح (حسب الطابع الزمني). |
| **Manual Resolution** | يتطلب تدخل المستخدم لحل التعارض. |
| **Merge** | دمج البيانات من كلا الطرفين (عند الإمكان). |

### 3.3 استراتيجيات التزامن (Sync Strategies)
| الاستراتيجية | الوصف |
| :--- | :--- |
| **تزامن كامل (Full Sync)** | مزامنة جميع البيانات (مناسب للبيانات الصغيرة أو التزامن الأولي). |
| **تزامن تدريجي (Incremental Sync)** | مزامنة البيانات التي تغيرت فقط منذ آخر مزامنة (مناسب للبيانات الكبيرة). |
| **تزامن عند الطلب (On-Demand Sync)** | مزامنة البيانات عند طلب المستخدم. |
| **تزامن دوري (Periodic Sync)** | مزامنة البيانات بشكل دوري (كل 5 دقائق، كل ساعة، الخ). |
| **تزامن فوري (Real-time Sync)** | مزامنة البيانات فوراً عند توفر الاتصال. |

## 4. إدارة البيانات المحلية (Local Data Management)

### 4.1 هيكل البيانات المحلية
| الجدول | الوصف |
| :--- | :--- |
| **المستخدمون (Users)** | بيانات المستخدمين المخزنة محلياً (للتحقق من الصلاحيات). |
| **المنتجات (Products)** | بيانات المنتجات المخزنة محلياً (للبحث والإضافة للعربة). |
| **العملاء (Customers)** | بيانات العملاء المخزنة محلياً (لإنشاء الفواتير). |
| **الفواتير (Invoices)** | الفواتير التي تم إنشاؤها في وضع الأوفلاين (بانتظار المزامنة). |
| **المخزون (Inventory)** | بيانات المخزون المحلية (للتحقق من التوفر). |
| **الورديات (Shifts)** | بيانات ورديات الكاشير المخزنة محلياً. |
| **المزامنة (Sync Queue)** | طابور العمليات التي تنتظر المزامنة. |

### 4.2 قواعد البيانات المحلية
- **التخزين المحدود:** تحديد الحد الأقصى لحجم البيانات المخزنة محلياً (مثلاً: 500 ميجابايت).
- **تنظيف البيانات:** حذف البيانات القديمة أو غير الضرورية تلقائياً عند تجاوز الحد الأقصى.
- **النسخ الاحتياطي:** عمل نسخ احتياطي للبيانات المحلية بشكل دوري.
- **التشفير:** تشفير البيانات المحلية لحمايتها من الوصول غير المصرح به.
- **التحقق من الصلاحية:** التحقق من صلاحية البيانات المحلية قبل استخدامها (مثل: التحقق من تواريخ انتهاء الصلاحية).

## 5. طابور المزامنة (Sync Queue)

### 5.1 هيكل طابور المزامنة
| الحقل | الوصف | إلزامي |
| :--- | :--- | :--- |
| **رقم العملية (Operation ID)** | رقم فريد للعملية. | نعم |
| **نوع العملية (Operation Type)** | إدراج (Insert)، تحديث (Update)، حذف (Delete). | نعم |
| **الجدول (Table)** | اسم الجدول المرتبط بالعملية. | نعم |
| **المفتاح الأساسي (Primary Key)** | المفتاح الأساسي للسجل. | نعم |
| **البيانات (Data)** | البيانات الكاملة للسجل (بتنسيق JSON). | نعم |
| **الطابع الزمني (Timestamp)** | وقت إنشاء العملية. | نعم |
| **حالة المزامنة (Sync Status)** | معلق، قيد المزامنة، مزامن، فشل. | نعم |
| **عدد المحاولات (Retry Count)** | عدد محاولات المزامنة الفاشلة. | نعم |
| **سبب الفشل (Failure Reason)** | سبب فشل المزامنة (إن وجد). | لا |

### 5.2 إدارة طابور المزامنة
- **إضافة العمليات:** إضافة العمليات إلى الطابور عند تنفيذها في وضع الأوفلاين.
- **معالجة العمليات:** معالجة العمليات في الطابور حسب الترتيب (FIFO) أو حسب الأولوية.
- **إعادة المحاولة:** إعادة محاولة العمليات الفاشلة تلقائياً (مع زيادة الفترة الزمنية بين المحاولات).
- **الحد الأقصى للمحاولات:** تحديد الحد الأقصى لعدد محاولات إعادة المزامنة (مثلاً: 5 محاولات).
- **تنظيف الطابور:** حذف العمليات القديمة أو الناجحة تلقائياً.

## 6. معالجة التعارضات (Conflict Resolution)

### 6.1 أنواع التعارضات
| النوع | الوصف | مثال |
| :--- | :--- | :--- |
| **تضارب الإدراج (Insert-Insert)** | إدراج نفس السجل في جهازين مختلفين. | إضافة نفس العميل في جهازين. |
| **تضارب التحديث (Update-Update)** | تحديث نفس السجل في جهازين مختلفين. | تغيير سعر منتج في جهازين. |
| **تضارب الحذف (Delete-Update)** | حذف سجل في جهاز وتحديثه في جهاز آخر. | حذف منتج في جهاز وتحديث سعره في آخر. |
| **تضارب الإدراج-التحديث (Insert-Update)** | إدراج سجل في جهاز وتحديث سجل مشابه في آخر. | إضافة عميل جديد وتحديث عميل موجود بنفس المعرف. |

### 6.2 استراتيجيات حل التعارضات
| الاستراتيجية | الوصف | التطبيق |
| :--- | :--- | :--- |
| **Server-Wins** | بيانات الخادم هي الأصح. | البيانات الأساسية (المنتجات، الأسعار، المستخدمين). |
| **Client-Wins** | بيانات العميل هي الأصح. | المبيعات، الفواتير، الحضور. |
| **Last-Write-Wins** | التعديل الأحدث هو الأصح. | الملاحظات، التعليقات، التحديثات غير الحرجة. |
| **Manual Resolution** | يتطلب تدخل المستخدم. | التعارضات المعقدة أو الحساسة. |
| **Merge** | دمج البيانات من كلا الطرفين. | عندما يكون الدمج ممكناً (مثل: إضافة حقول مختلفة). |
| **Ignore** | تجاهل التغيير من أحد الطرفين. | عندما يكون التغيير غير مهم. |

### 6.3 سجل التعارضات (Conflict Log)
| الحقل | الوصف |
| :--- | :--- |
| **رقم التعارض (Conflict ID)** | رقم فريد للتعارض. |
| **التاريخ (Date)** | تاريخ حدوث التعارض. |
| **الجدول (Table)** | الجدول المرتبط بالتعارض. |
| **المفتاح الأساسي (Primary Key)** | المفتاح الأساسي للسجل المتعارض. |
| **بيانات العميل (Client Data)** | البيانات من الجهاز المحلي. |
| **بيانات الخادم (Server Data)** | البيانات من الخادم المركزي. |
| **استراتيجية الحل (Resolution Strategy)** | الاستراتيجية المستخدمة لحل التعارض. |
| **الحالة (Status)** | محلول، قيد المعالجة، معلق. |
| **المستخدم (User)** | المستخدم الذي حل التعارض (إن كان يدوياً). |

## 7. سياسات المزامنة (Sync Policies)

### 7.1 سياسات المزامنة حسب الوحدة
| الوحدة | استراتيجية التعارض | تواتر المزامنة | حجم البيانات |
| :--- | :--- | :--- | :--- |
| **المبيعات (Sales)** | Client-Wins | فوري / دوري | متوسط |
| **المخزون (Inventory)** | Server-Wins | دوري (كل 5 دقائق) | كبير |
| **المنتجات (Products)** | Server-Wins | دوري (كل ساعة) | كبير |
| **العملاء (Customers)** | Server-Wins | دوري (كل ساعة) | متوسط |
| **المالية (Finance)** | Server-Wins | فوري | صغير |
| **الموارد البشرية (HR)** | Server-Wins | دوري (يومي) | متوسط |
| **المشتريات (Procurement)** | Server-Wins | فوري | متوسط |
| **المستخدمين (Users)** | Server-Wins | دوري (يومي) | صغير |
| **الورديات (Shifts)** | Client-Wins | فوري | صغير |

### 7.2 سياسات التخزين المحلي
| السياسة | الوصف |
| :--- | :--- |
| **الحد الأقصى لحجم البيانات** | تحديد الحد الأقصى لحجم البيانات المخزنة محلياً (مثلاً: 1 جيجابايت). |
| **فترة الاحتفاظ** | تحديد المدة التي يتم فيها الاحتفاظ بالبيانات محلياً (مثلاً: 30 يوماً). |
| **تنظيف البيانات** | حذف البيانات القديمة تلقائياً عند تجاوز الحد الأقصى أو انتهاء فترة الاحتفاظ. |
| **البيانات الأساسية** | تحديد البيانات التي يجب الاحتفاظ بها دائماً (مثل: المنتجات النشطة، العملاء النشطين). |

## 8. سلوك الأوفلاين حسب الوحدة (Offline Behavior by Module)

### 8.1 المبيعات ونقاط البيع (Sales & POS)
- **المسموح به في الأوفلاين:**
    - البحث عن المنتجات وإضافتها للسلة.
    - إنشاء فواتير المبيعات.
    - طباعة الإيصالات.
    - إدارة الورديات (فتح، تعليق، إغلاق).
    - استلام المدفوعات النقدية.
    - إنشاء مرتجعات (مع ربط بالفاتورة الأصلية).
- **غير المسموح به في الأوفلاين:**
    - التحقق من الائتمان للعملاء.
    - الدفع بالبطاقة (تحتاج اتصال بالبنك).
    - التحقق من المخزون المركزي (يتم الاعتماد على المخزون المحلي).
- **المزامنة:**
    - رفع الفواتير والمرتجعات والورديات.
    - Client-Wins للمبيعات.

### 8.2 المخزون (Inventory)
- **المسموح به في الأوفلاين:**
    - عرض بيانات المخزون المحلية.
    - إجراء جرد باستخدام أجهزة PDA.
    - تسجيل حركات المخزون (استلام، صرف، تحويل) - مع تخزينها مؤقتاً.
- **غير المسموح به في الأوفلاين:**
    - تحديث التكلفة (Moving Average) - يتم بواسطة الخادم.
    - التحقق من المخزون في فروع أخرى.
- **المزامنة:**
    - رفع حركات المخزون ونتائج الجرد.
    - تنزيل تحديثات التكلفة والأرصدة المركزية.
    - Client-Wins للجرد والحركات المحلية؛ Server-Wins للتكلفة.

### 8.3 الموارد البشرية (HR)
- **المسموح به في الأوفلاين:**
    - عرض بيانات الموظفين المخزنة محلياً.
    - تسجيل الحضور والانصراف.
    - تقديم طلبات الإجازات (تخزينها مؤقتاً).
- **غير المسموح به في الأوفلاين:**
    - حساب الرواتب.
    - تحديث بيانات الموظفين الأساسية.
    - الموافقة على الإجازات (تخزن طلبات الموافقة).
- **المزامنة:**
    - رفع الحضور، طلبات الإجازات، تحديثات البيانات.
    - تنزيل تحديثات بيانات الموظفين، سياسات الإجازات.
    - Client-Wins للحضور وطلبات الإجازات؛ Server-Wins للبيانات الأساسية.

### 8.4 المشتريات (Procurement)
- **المسموح به في الأوفلاين:**
    - إنشاء طلبات الشراء (تخزينها مؤقتاً).
    - إنشاء أوامر الشراء (تخزينها مؤقتاً).
    - استلام البضاعة (تسجيل الاستلام محلياً).
- **غير المسموح به في الأوفلاين:**
    - التحقق من الميزانيات.
    - الموافقة على أوامر الشراء (تخزن طلبات الموافقة).
- **المزامنة:**
    - رفع طلبات وأوامر الشراء، استلام البضاعة.
    - تنزيل تحديثات الموردين، الأسعار، الموافقات.
    - Server-Wins للموافقات؛ Client-Wins للاستلام.

### 8.5 المالية (Finance)
- **المسموح به في الأوفلاين:**
    - عرض البيانات المالية المخزنة محلياً (للقراءة فقط).
    - إنشاء قيود يومية مؤقتة (تنتظر المزامنة).
- **غير المسموح به في الأوفلاين:**
    - ترحيل القيود المحاسبية.
    - إصدار التقارير المالية (قد تكون غير محدثة).
    - إغلاق الفترات المالية.
- **المزامنة:**
    - رفع القيود اليومية المؤقتة.
    - تنزيل الترحيلات والتسويات.
    - Server-Wins (البيانات المركزية هي الأصح).

### 8.6 إدارة المستندات (Document Management)
- **المسموح به في الأوفلاين:**
    - عرض المستندات المخزنة محلياً.
    - تحميل المستندات المخزنة محلياً.
    - رفع مستندات جديدة (تخزينها مؤقتاً).
- **غير المسموح به في الأوفلاين:**
    - مشاركة المستندات.
    - الموافقة على المستندات.
- **المزامنة:**
    - رفع المستندات الجديدة.
    - تنزيل المستندات الجديدة والمحدثة.
    - Server-Wins (النسخة الأحدث في الخادم).

## 9. استرداد البيانات (Data Recovery)

### 9.1 سيناريوهات الاسترداد
| السيناريو | آلية الاسترداد |
| :--- | :--- |
| **فشل المزامنة الجزئي** | إعادة محاولة المزامنة للبيانات الفاشلة فقط. |
| **فشل المزامنة الكامل** | إعادة محاولة المزامنة لجميع البيانات في الطابور. |
| **فقدان البيانات المحلية** | استعادة البيانات من الخادم المركزي (تنزيل كامل للبيانات). |
| **تلف البيانات المحلية** | استعادة البيانات من النسخ الاحتياطي المحلي (إن وجد) أو من الخادم. |
| **تعارض غير قابل للحل** | إشعار مسؤول النظام للتدخل اليدوي. |

### 9.2 نقاط الاستعادة (Recovery Points)
- **نقطة المزامنة الأخيرة:** استعادة البيانات إلى آخر نقطة مزامنة ناجحة.
- **نقطة التخزين المؤقت:** استعادة البيانات من ذاكرة التخزين المؤقت.
- **نقطة النسخ الاحتياطي:** استعادة البيانات من النسخة الاحتياطية المحلية.

---

## 10. مراقبة المزامنة (Sync Monitoring)

### 10.1 مؤشرات الأداء (KPIs)
| المؤشر | الوصف |
| :--- | :--- |
| **معدل نجاح المزامنة (Sync Success Rate)** | نسبة العمليات التي تمت مزامنتها بنجاح. |
| **زمن المزامنة (Sync Duration)** | متوسط الوقت المستغرق لعملية المزامنة. |
| **حجم البيانات المنقولة (Data Volume)** | حجم البيانات المنقولة أثناء المزامنة. |
| **عدد التعارضات (Conflict Count)** | عدد التعارضات التي حدثت أثناء المزامنة. |
| **حالة طابور المزامنة (Queue Status)** | عدد العمليات في طابور المزامنة (معلقة، قيد المزامنة، فاشلة). |

### 10.2 لوحات المعلومات (Dashboards)
- **لوحة حالة المزامنة:** عرض حالة المزامنة الحالية (متصل، غير متصل، قيد المزامنة).
- **لوحة أداء المزامنة:** عرض مؤشرات الأداء (معدل النجاح، زمن المزامنة).
- **لوحة التعارضات:** عرض التعارضات وحالاتها.

### 10.3 التنبيهات (Alerts)
- **فشل المزامنة:** تنبيه عند فشل المزامنة لأكثر من X محاولات.
- **تجاوز حجم البيانات:** تنبيه عند تجاوز الحد الأقصى لحجم البيانات المحلية.
- **تضارب كبير:** تنبيه عند حدوث تعارض يتطلب تدخل يدوي.
- **انقطاع الاتصال:** تنبيه عند انقطاع الاتصال بالخادم المركزي.

---

## 11. التوسعات المستقبلية (Future Extensions)

- **المزامنة الذكية (Smart Sync):** استخدام الذكاء الاصطناعي لتحديد أولويات المزامنة وتقليل التعارضات.
- **المزامنة التكيفية (Adaptive Sync):** تغيير استراتيجية المزامنة تلقائياً بناءً على حالة الشبكة وحجم البيانات.
- **المزامنة اللامركزية (Decentralized Sync):** دعم المزامنة بين الأجهزة مباشرة (بدون خادم مركزي) في بعض السيناريوهات.
- **التنبؤ بانقطاع الاتصال:** التنبؤ بانقطاع الاتصال مسبقاً والاستعداد له (تنزيل البيانات المهمة مسبقاً).

---

## 12. متطلبات الأداء (Performance Requirements)

- **تبديل وضع الأوفلاين:** أقل من 1 ثانية (دون فقدان البيانات أو تعطيل المستخدم).
- **المزامنة:** مزامنة 1,000 عملية في أقل من 30 ثانية.
- **تخزين البيانات المحلية:** دعم تخزين ما يصل إلى 500 ميجابايت من البيانات.
- **البحث المحلي:** البحث في البيانات المحلية في أقل من 200 مللي ثانية.

## 13. سيناريوهات الإجهاد المؤسسية (Enterprise Stress Scenarios)

| السيناريو | آلية المواجهة |
| :--- | :--- |
| **انقطاع الاتصال لعدة ساعات** | تخزين جميع العمليات محلياً، المزامنة عند عودة الاتصال. |
| **آلاف الأجهزة المتصلة في نفس الوقت** | استخدام خوادم متعددة للمزامنة، توزيع الحمل. |
| **فشل الخادم المركزي** | استمرار العمل في وضع الأوفلاين، المزامنة عند عودة الخادم. |
| **بيانات ضخمة في وضع الأوفلاين** | ضغط البيانات، التزامن التدريجي (Incremental Sync). |
| **تعارضات متعددة في نفس الوقت** | استخدام استراتيجيات حل تعارضات تلقائية، إشعار المدير في الحالات المعقدة. |
