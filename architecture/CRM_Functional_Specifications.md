# المواصفات الوظيفية المؤسسية لوحدة إدارة علاقات العملاء (CRM)
**المشروع:** Smart Merchant ERP  
**النطاق:** Customer Relationship Management Module  
**الإصدار:** 2.0 (النسخة المحسّنة والموحّدة)  
**الهدف:** توثيق شامل ومتكامل لكافة متطلبات، عمليات، سياسات، وقيود وحدة إدارة علاقات العملاء، لتكون مرجعاً هندسياً وتشغيلياً مغلقاً للمطورين ومحللي الأعمال.

---

## 1. الأهداف الإستراتيجية والنطاق

### 1.1 الأهداف
- توفير رؤية 360 درجة للعميل تشمل جميع تفاعلاته مع الشركة (مبيعات، دعم، تواصل، تفضيلات).
- تمكين فرق المبيعات والتسويق والدعم من إدارة علاقات العملاء بفعالية وكفاءة.
- زيادة ولاء العملاء وتحسين تجربتهم من خلال التفاعلات الشخصية والاستباقية.
- دعم عمليات البيع الإضافية (Upselling) والبيع التكميلي (Cross-selling) لزيادة الإيرادات.
- توفير تحليلات متقدمة لفهم سلوك العملاء وتوقع احتياجاتهم.

### 1.2 النطاق
يشمل النطاق إدارة بيانات العملاء الأساسية، إدارة التفاعلات (المكالمات، البريد الإلكتروني، الاجتماعات، المهام)، إدارة شكاوى العملاء وطلبات الدعم، إدارة فرص البيع، إدارة الحملات التسويقية، وإدارة برامج الولاء.

### 1.3 المسؤوليات الأساسية
- **إدارة بيانات العملاء:** الحفاظ على دقة واكتمال بيانات العملاء.
- **تحسين تجربة العميل:** ضمان تجربة إيجابية ومتسقة عبر جميع نقاط الاتصال.
- **زيادة الإيرادات:** تحديد فرص البيع الإضافي والبيع التكميلي.
- **تحليل العملاء:** توفير رؤى قابلة للتنفيذ حول سلوك العملاء وتفضيلاتهم.
- **التكامل:** التكامل مع وحدات المبيعات والتسويق والدعم والمالية.

## 2. الأمان والصلاحيات (الحوكمة)

### 2.1 أدوار المستخدمين (User Roles)
| الدور | المسؤولية الأساسية |
| :--- | :--- |
| **مندوب المبيعات (Sales Rep)** | إدارة العملاء، متابعة فرص البيع، تسجيل التفاعلات. |
| **مدير المبيعات (Sales Manager)** | الإشراف على فريق المبيعات، تحليل الأداء، اعتماد الصفقات الكبيرة. |
| **أخصائي الدعم (Support Agent)** | معالجة شكاوى العملاء وطلبات الدعم. |
| **مدير الدعم (Support Manager)** | الإشراف على فريق الدعم، تحليل رضا العملاء. |
| **أخصائي التسويق (Marketing Specialist)** | إنشاء وإدارة الحملات التسويقية، تحليل التفاعلات. |
| **مدير العلاقات (CRM Manager)** | الإدارة العامة لوحدة CRM، سياسات البيانات، التدريب. |

### 2.2 الصلاحيات الأساسية (Permissions)
- `View_Customer` (عرض بيانات العميل)
- `Create_Customer` (إنشاء عميل جديد)
- `Edit_Customer` (تعديل بيانات العميل)
- `Delete_Customer` (حذف عميل - مقيد جداً)
- `Merge_Customer` (دمج عميلين متكررين)
- `View_Opportunity` (عرض فرص البيع)
- `Manage_Opportunity` (إدارة فرص البيع - إنشاء، تعديل، إغلاق)
- `Manage_Ticket` (إدارة طلبات الدعم)
- `Manage_Campaign` (إدارة الحملات التسويقية)
- `View_Sensitive_Data` (عرض بيانات حساسة مثل معلومات الدفع)

### 2.3 القواعد الأمنية (Security Rules)
- **منع حذف العملاء:** يمنع حذف أي عميل له سجل مبيعات أو تفاعلات. بدلاً من ذلك، يتم تغيير حالة العميل إلى "غير نشط" أو "محظور".
- **تشفير البيانات الحساسة:** تشفير المعلومات الحساسة مثل أرقام الهوية، معلومات الدفع.
- **الموافقة على البيانات الشخصية:** الالتزام بقوانين حماية البيانات (مثل GDPR) والحصول على موافقة العميل على تخزين ومعالجة بياناته.
- **سجل التدقيق:** تسجيل جميع التغييرات على بيانات العميل (من أنشأ، متى، ماذا غيّر).
- **الوصول على أساس الحاجة (Need-to-Know):** الوصول إلى بيانات العملاء يكون فقط للمستخدمين الذين يحتاجونها لأداء مهامهم.

### 3.2 تصنيف العملاء (Customer Classification)
| التصنيف | الوصف | المعايير |
| :--- | :--- | :--- |
| **VIP** | عملاء مميزون ذوو إنفاق عالٍ وولاء قوي. | حجم المشتريات، تكرار الشراء، التفاعل الإيجابي. |
| **ذهبي (Gold)** | عملاء ذوو إنفاق مرتفع. | حجم المشتريات > حد معين. |
| **فضي (Silver)** | عملاء ذوو إنفاق متوسط. | حجم المشتريات بين حدين. |
| **برونزي (Bronze)** | عملاء جدد أو ذوو إنفاق منخفض. | حجم المشتريات < حد معين. |
| **محتمل (Lead)** | عميل محتمل لم يشتري بعد. | لم يقم بأي عملية شراء. |
| **محظور (Blocked)** | عميل محظور بسبب مخالفات أو احتيال. | مخالفات متكررة، شكاوى، احتيال. |

### 3.3 دمج العملاء (Customer Merge)
- **الغرض:** دمج حسابات مكررة لنفس العميل.
- **آلية الدمج:**
    1.  تحديد الحساب الرئيسي (الحساب الذي سيتم الاحتفاظ به).
    2.  دمج جميع السجلات (المبيعات، التفاعلات، الدعم) في الحساب الرئيسي.
    3.  إلغاء تنشيط الحسابات المدمجة.
- **قواعد الدمج:**
    - يتطلب موافقة مدير CRM.
    - تسجيل عملية الدمج في سجل التدقيق.
    - إرسال إشعار للعميل (اختياري).

## 4. إدارة التفاعلات (Interaction Management)

### 4.1 أنواع التفاعلات (Interaction Types)
| النوع | الوصف |
| :--- | :--- |
| **مكالمة هاتفية (Phone Call)** | مكالمة هاتفية مع العميل (صادرة أو واردة). |
| **بريد إلكتروني (Email)** | مراسلة إلكترونية مع العميل. |
| **اجتماع (Meeting)** | اجتماع شخصي أو افتراضي مع العميل. |
| **رسالة نصية (SMS)** | رسالة نصية قصيرة. |
| **تذكير (Reminder)** | تذكير لمتابعة العميل. |
| **ملاحظة (Note)** | ملاحظة عامة عن العميل. |
| **مهمة (Task)** | مهمة يجب تنفيذها للعميل. |
| **تفاعل وسائل التواصل الاجتماعي (Social Media)** | تفاعل على منصات التواصل الاجتماعي. |
| **زيارة موقع (Website Visit)** | تتبع زيارة العميل للموقع الإلكتروني. |

### 4.2 تسجيل التفاعلات (Logging Interactions)
- **المحتوى:**
    - نوع التفاعل.
    - التاريخ والوقت.
    - الملخص (Summary).
    - التفاصيل الكاملة (Details).
    - المستخدم المسؤول.
    - المستندات المرفقة (إن وجدت).
    - الحالة (مغلقة، مفتوحة، قيد المتابعة).
- **قواعد التسجيل:**
    - تسجيل التفاعل تلقائياً (عند إجراء مكالمة، إرسال بريد إلكتروني).
    - تسجيل يدوي (للتفاعلات غير الرقمية).
    - ربط التفاعل بفرصة بيع أو تذكرة دعم (إن وجدت).

### 4.3 متابعة العملاء (Follow-up)
- **التذكيرات التلقائية:** تنبيه تلقائي لمندوب المبيعات بمتابعة العميل في تاريخ محدد.
- **سير العمل التلقائي:** إنشاء مهمة متابعة تلقائياً بعد تفاعل معين (مثلاً: بعد إرسال عرض سعر).
- **قواعد المتابعة:**
    - متابعة العملاء المحتملين خلال 24 ساعة من أول تواصل.
    - متابعة العملاء الحاليين بشكل دوري (حسب تصنيفهم).
    - متابعة العملاء بعد عملية شراء (لضمان رضاهم).

## 5. إدارة فرص البيع (Opportunity Management)

### 5.1 دورة حياة الفرصة (Opportunity Lifecycle)
1.  **تحديد الفرصة (Identification):** تحديد عميل محتمل أو حاجة جديدة لعميل حالي.
2.  **التأهيل (Qualification):** تقييم الفرصة من حيث القيمة، الاحتمالية، والموارد المطلوبة.
3.  **التطوير (Development):** بناء علاقة مع العميل، تقديم عروض، مناقشة الاحتياجات.
4.  **التفاوض (Negotiation):** التفاوض على السعر والشروط.
5.  **الإغلاق (Closing):** إتمام الصفقة (تحويل الفرصة إلى أمر بيع أو فاتورة).
6.  **المتابعة (Follow-up):** ضمان رضا العميل، تحديد فرص مستقبلية.

### 5.2 بيانات الفرصة (Opportunity Data)
| الحقل | الوصف | إلزامي |
| :--- | :--- | :--- |
| **الاسم (Name)** | اسم الفرصة (مثل "بيع 100 جهاز لشركة X"). | نعم |
| **العميل (Customer)** | العميل المرتبط بالفرصة. | نعم |
| **القيمة المتوقعة (Expected Value)** | القيمة المالية المتوقعة للصفقة. | نعم |
| **احتمالية الإغلاق (Probability)** | نسبة احتمالية إتمام الصفقة (0-100%). | نعم |
| **مرحلة الفرصة (Stage)** | المرحلة الحالية من دورة حياة الفرصة. | نعم |
| **تاريخ الإغلاق المتوقع (Expected Close Date)** | التاريخ المتوقع لإتمام الصفقة. | نعم |
| **المصدر (Source)** | كيف تم تحديد الفرصة (إحالة، تسويق، مبيعات). | لا |
| **المنافسون (Competitors)** | المنافسون في هذه الصفقة. | لا |
| **المنتجات/الخدمات (Products/Services)** | المنتجات أو الخدمات المعروضة. | نعم |
| **المستخدم المسؤول (Owner)** | مندوب المبيعات المسؤول عن الفرصة. | نعم |

### 5.3 قواعد إدارة الفرص
- **تحديث الفرص:** تحديث حالة وقيمة الفرص بشكل دوري.
- **تنبيهات الفرص:** تنبيه عند اقتراب تاريخ الإغلاق المتوقع.
- **فرص متداخلة:** منع إنشاء فرص متداخلة لنفس العميل ونفس المنتج.
- **ترقية الفرص:** ترقية الفرصة تلقائياً إلى أمر بيع عند الإغلاق الناجح.
- **إغلاق الفرص:** إغلاق الفرصة كـ "رابحة" أو "خاسرة" مع تسجيل السبب.

## 6. إدارة شكاوى العملاء وطلبات الدعم (Support & Complaints)

### 6.1 دورة حياة التذكرة (Ticket Lifecycle)
1.  **إنشاء التذكرة (Create):** تسجيل شكوى أو طلب دعم من العميل.
2.  **التصنيف (Categorize):** تصنيف التذكرة حسب النوع، الأولوية، الفئة.
3.  **التخصيص (Assign):** تخصيص التذكرة لفريق الدعم المناسب.
4.  **المعالجة (Process):** معالجة الشكوى أو الطلب.
5.  **الحل (Resolve):** تقديم حل للعميل.
6.  **الإغلاق (Close):** إغلاق التذكرة بعد تأكيد العميل على الحل.

### 6.2 بيانات التذكرة (Ticket Data)
| الحقل | الوصف | إلزامي |
| :--- | :--- | :--- |
| **الرقم (Number)** | رقم فريد للتذكرة. | نعم |
| **العميل (Customer)** | العميل صاحب الشكوى. | نعم |
| **الموضوع (Subject)** | عنوان مختصر للتذكرة. | نعم |
| **الوصف (Description)** | وصف تفصيلي للشكوى أو الطلب. | نعم |
| **النوع (Type)** | شكوى، طلب دعم، استفسار، اقتراح. | نعم |
| **الأولوية (Priority)** | عالية، متوسطة، منخفضة. | نعم |
| **الفئة (Category)** | فنية، خدمية، محاسبية، شحن. | لا |
| **الحالة (Status)** | مفتوحة، قيد المعالجة، في انتظار الرد، مغلقة. | نعم |
| **المستخدم المسؤول (Assignee)** | موظف الدعم المسؤول. | نعم |
| **المصدر (Source)** | هاتف، بريد إلكتروني، موقع، تطبيق. | لا |

### 6.3 قواعد الدعم
- **الاستجابة السريعة:** الاستجابة للتذاكر عالية الأولوية خلال 4 ساعات، المتوسطة خلال 24 ساعة.
- **التصعيد (Escalation):** تصعيد التذاكر غير المحلولة إلى مدير الدعم بعد فترة محددة.
- **رضا العميل:** طلب تقييم من العميل بعد إغلاق التذكرة.
- **تحليل الأسباب:** تحليل أسباب الشكاوى المتكررة لمعالجتها جذرياً.

### 7.3 قواعد الحملات

- **تخصيص الرسائل:** تخصيص محتوى الحملة بناءً على شريحة العميل.
- **تتبع الأداء:** تتبع معدل الفتح، النقر، التحويل لكل حملة.
- **تحليل العائد على الاستثمار (ROI):** حساب العائد على الاستثمار لكل حملة.

## 8. برامج الولاء (Loyalty Programs)

### 8.1 أنواع برامج الولاء
| النوع | الوصف |
| :--- | :--- |
| **نقاط الولاء (Points)** | تراكم نقاط مقابل كل عملية شراء، وتحويلها إلى خصومات أو منتجات مجانية. |
| **المستويات (Tiers)** | مستويات متعددة (ذهبي، فضي، برونزي) لكل مستوى مزايا مختلفة. |
| **الاشتراكات (Subscriptions)** | اشتراكات مدفوعة تمنح مزايا حصرية (شحن مجاني، خصومات خاصة). |
| **الكاش باك (Cashback)** | استرداد جزء من قيمة المشتريات كرصيد. |
| **العروض الحصرية (Exclusive Offers)** | عروض وخصومات حصرية لأعضاء برنامج الولاء. |

### 8.2 بيانات برنامج الولاء
| الحقل | الوصف |
| :--- | :--- |
| **اسم البرنامج (Program Name)** | اسم برنامج الولاء. |
| **نوع البرنامج (Type)** | نقاط، مستويات، اشتراكات. |
| **قواعد تراكم النقاط (Earning Rules)** | كم نقطة لكل ريال يتم إنفاقه. |
| **قواعد استبدال النقاط (Redeeming Rules)** | كم نقطة مطلوبة للحصول على خصم أو منتج. |
| **مستويات البرنامج (Tiers)** | المستويات والمزايا لكل مستوى. |
| **تاريخ البدء (Start Date)** | تاريخ بدء البرنامج. |
| **تاريخ الانتهاء (End Date)** | تاريخ انتهاء البرنامج. |
| **الحالة (Status)** | نشط، غير نشط. |

### 8.3 قواعد برامج الولاء
- **تراكم النقاط:** تراكم النقاط تلقائياً عند إتمام عملية الشراء.
- **استبدال النقاط:** استبدال النقاط بخصم أو منتج عند الطلب.
- **انتهاء النقاط:** انتهاء صلاحية النقاط بعد فترة محددة (إشعار العميل قبل الانتهاء).
- **ترقية المستوى:** ترقية العميل تلقائياً إلى مستوى أعلى عند تحقيق الشروط.

---

## 9. العمل في وضع الأوفلاين (Offline Mode)

### 9.1 السلوك في وضع الأوفلاين
- **عرض العملاء:** عرض بيانات العملاء المخزنة محلياً.
- **إضافة العملاء:** إضافة عملاء جدد (يتم المزامنة عند عودة الاتصال).
- **تسجيل التفاعلات:** تسجيل التفاعلات محلياً.
- **إدارة المهام:** إنشاء وتحديث المهام محلياً.
- **تسجيل التذاكر:** إنشاء تذاكر الدعم محلياً.

### 9.2 المزامنة (Synchronization)
- **رفع البيانات:** رفع العملاء الجدد، التفاعلات، المهام، التذاكر.
- **تنزيل البيانات:** تنزيل تحديثات بيانات العملاء من الخادم المركزي.
- **سياسة النزاعات:** Client-Wins للتفاعلات والمهام؛ Server-Wins لبيانات العملاء الأساسية.

## 10. التقارير ولوحات المعلومات (Analytics & Reporting)

### 10.1 التقارير الأساسية
| التقرير | الغرض |
| :--- | :--- |
| **تقرير العملاء (Customer Report)** | عرض كافة العملاء مع بياناتهم وتصنيفاتهم. |
| **تقرير تفاعلات العملاء (Interaction Report)** | عرض جميع التفاعلات مع العملاء. |
| **تقرير فرص البيع (Opportunity Report)** | عرض فرص البيع حسب المرحلة، القيمة، الاحتمالية. |
| **تقرير أداء المبيعات (Sales Performance)** | أداء مندوبي المبيعات (عدد الصفقات، القيمة). |
| **تقرير شكاوى العملاء (Complaints Report)** | تحليل الشكاوى حسب النوع، الأولوية، التصنيف. |
| **تقرير رضا العملاء (Satisfaction Report)** | نتائج استبيانات رضا العملاء. |
| **تقرير الحملات التسويقية (Campaign Report)** | أداء الحملات (الوصول، المشاركة، التحويل). |
| **تقرير برامج الولاء (Loyalty Report)** | استخدام برامج الولاء، النقاط المكتسبة والمستخدمة. |

### 10.2 لوحات المعلومات (Dashboards)
- **لوحة CRM الرئيسية:** عدد العملاء، عدد الفرص، عدد التذاكر، رضا العملاء.
- **لوحة أداء المبيعات:** المبيعات حسب مندوب المبيعات، حسب المنطقة، حسب المنتج.
- **لوحة دعم العملاء:** عدد التذاكر المفتوحة، وقت الحل، رضا العملاء.
- **لوحة الحملات التسويقية:** أداء الحملات، العائد على الاستثمار.
- **لوحة ولاء العملاء:** عدد الأعضاء، النقاط المكتسبة والمستخدمة.

### 10.3 مؤشرات الأداء (KPIs)
| المؤشر | الوصف |
| :--- | :--- |
| **معدل الاحتفاظ بالعملاء (Retention Rate)** | نسبة العملاء الذين استمروا في الشراء. |
| **القيمة الدائمة للعميل (CLV)** | إجمالي الإيرادات المتوقعة من العميل طوال علاقته بالشركة. |
| **تكلفة اكتساب العميل (CAC)** | التكلفة الإجمالية لاكتساب عميل جديد. |
| **معدل التحويل (Conversion Rate)** | نسبة العملاء المحتملين الذين تحولوا إلى عملاء فعليين. |
| **رضا العملاء (CSAT)** | درجة رضا العملاء عن الخدمة. |
| **معدل حل التذاكر (Resolution Rate)** | نسبة التذاكر التي تم حلها. |
| **معدل المبيعات الإضافية (Upsell Rate)** | نسبة العملاء الذين اشتروا منتجات إضافية. |

## 11. التوسعات المستقبلية (Future Extensions)

- **الذكاء الاصطناعي (AI):** التنبؤ بسلوك العملاء، اقتراح المنتجات، تحليل المشاعر في التفاعلات.
- **روبوتات الدردشة (Chatbots):** دعم آلي للعملاء عبر الدردشة الفورية.
- **تحليل وسائل التواصل الاجتماعي (Social Listening):** تتبع وتحليل تفاعلات العملاء على وسائل التواصل الاجتماعي.
- **إدارة العملاء المتعددة اللغات:** دعم التواصل مع العملاء بلغات متعددة.
- **بوابة العميل (Customer Portal):** منصة ذاتية الخدمة للعملاء لعرض فواتيرهم، تتبع الطلبات، وفتح تذاكر الدعم.

- **تسجيل تفاعل:** أقل من 200 مللي ثانية.
- **إنشاء تذكرة:** أقل من ثانية.
